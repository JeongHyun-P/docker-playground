### Docker?

컨테이너를 사용하여 프로그램을 각각 분리된 환경에서 실행하고 관리

-   서비스 실행에 필요한 환경을 하나로 패키징
-   어떤 환경에서나 동일한 환경으로 서비스를 실행 및 배포가능
-   개발/운영 환경 차이를 최소화

### Container

-   컴퓨터 환경 내에서 독립적인 환경을 구성하여 프로그램을 별도로 실행할 수 있게 만든 개념
-   하나의 컨테이너 = 한 대의 독립적인 컴퓨터
-   컨테이너를 포함하는 실제 컴퓨터를 **호스트 컴퓨터**라고 부르고 컨테이너와 구분
-   각 컨테이너는 독립된 디스크(저장 공간)를 가지며, 서로의 자원에 접근할 수 없음
-   각 컨테이너는 고유의 네트워크(IP, PORT)를 보유

### Container vs Virtual Machine

|              | Container             | Virtual Machine     | note |
| ------------ | --------------------- | ------------------- | ---- |
| OS 포함 여부 | ❌ (호스트 커널 공유) | ⭕ (완전한 OS 포함) |
| 이미지 크기 | 작음 (MB 단위) | 큼 (GB 단위) |
| 실행 속도 | 빠름 (초 단위) | 느림 (분 단위) |
| 자원 사용량 | 적음 | 많음 | 도커 컨테이너는 호스트 OS의 커널을 공유하고 필요한 구성만 포함해 프로세스 단위로 실행되기 때문에 독립적인 OS를 포함하는 가상머신보다 메모리와 디스크 사용량이 훨씬 적음 |
| 보안성 | 상대적으로 낮음 | 높음 | 도커 컨테이너는 호스트 OS의 커널을 공유하므로 상대적으로 보안이 취약 |

### Image

-   프로그램(예: MySQL)을 실행하는데 필요한 설치 파일, 설정, 버전 정보 등을 포함하는 패키지
-   이미지 다운로드시 도커허브에서 다운받게됨
-   이미지의 태그명은 버전,환경,목적 등을 구분 하는 라벨 (기본적으로는 latest)

### Volume

볼륨은 컨테이너의 데이터를 영속적으로 저장하기 위한 방법이다.  
컨테이너는 기능이 추가되거나 수정사항이 생기면 기존 컨테이너 내부에서 수정하지 않고 새로운 컨테이너를 만들어서 통째로 갈아끼우는 방식을 사용한다.  
따라서 기존 컨테이너를 새로운 컨테이너로 교체하면 기존 컨테이너 내부 데이터도 같이 삭제되는데 내부 데이터가 삭제되면 안되는경우 볼륨을 활용해야한다.  
컨테이너 자체의 저장 공간을 사용하지 않고, 호스트 자체의 저장공간을 공유해서 사용하는 형태이다.  
호스트 디렉터리가 존재하면 컨테이너 디렉토리를 덮어쓰므로 볼륨 사용시 빈 디렉터리를 권장.

### 이미지 명령어
| 명령어 | 설명 | note |
| ---- | --- | --- |
| ```docker pull {image}``` | 도커 이미지 다운로드 | https://hub.docker.com 에서 이미지 정보 확인 |
| ```docker image ls``` | 다운받은 이미지 목록 |  |
| ```docker rm {image}``` | 이미지 삭제 | 삭제시 이미지ID의 일부만 입력해도 삭제가능. 컨테이너에서 사용중일땐 삭제 불가능함. |
| ```docker rm -f {image}``` | 이미지 강제 삭제 | 중단된 컨테이너일 경우 강제 삭제 가능하지만 실행중인 컨테이너일 경우 컨테이너를 중단 후 삭제가능함. |
| ```docker image rm $(docker images -q)``` | 컨테이너에서 사용되지 않고있는 이미지를 전부 삭제 | |

### 컨테이너 명령어
| 명령어 | 설명 | note |
| ---- | --- | --- |
| ```docker run ${image}``` | 이미지 다운 및 컨테이너 생성 및 실행 | foreground 동작 |
| ```docker run -d ${image}``` | 이미지 다운 및 컨테이너 생성 및 실행 | background 동작 |
| ```docker ps``` | 실행중인 컨테이너 조회 |  |
| ```docker ps -a``` | 전체 컨테이너 조회 (종료된 컨테이너 포함) |  |
| ```docker start ${Container ID}``` | 컨테이너 실행 |  |
| ```docker rm ${Container ID}``` | 컨테이너 삭제 | 삭제시 이미지ID의 일부만 입력해도 삭제가능, 여러 컨테이너 동시에 삭제 가능 |
| ```docker rm -f ${Container ID}``` | 컨테이너 강제 삭제 | 실행중인 컨테이너도 강제로 삭제 |
| ```docker rm $(docker ps -qa)``` | 중지된 컨테이너 전체 삭제 |  |
| ```docker run --name {name} -d {image}``` | 컨테이너 이름 지정 |  |
| ```docker run -d -p {Host PORT}:{Container PORT} {image}``` | 포트 매핑 |  |
| ```docker logs {Container ID}``` | 로그 확인 | ```--tail 10```: 마지막 10개의 로그만 확인 <br/> ```-f``` : 실시간 로그 확인 <br/> ```—tail 0 -f```: 이전로그 무시하고 실시간 로그 확인 |
| ```docker exec -it {Container ID} bash``` | 컨테이너 내부 접속 |  |
| ```docker run -v {Host Directory}:{Container Directory} {image}:{tag}```  | 볼륨 사용 | |
